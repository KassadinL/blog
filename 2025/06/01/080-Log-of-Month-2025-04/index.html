SICP

大名鼎鼎的 SICP，这个月看完前两章 Building Abstractions with Functions and Building Abstractions with Data

第一章上来就是 functional programming，全部递归写法
而且不使用 JavaScript 自带的数据类型，自己动手搓 pair(), head(), tail() 三件套

SICP 全部基于这种 function 里套 function 的写法

我看的是 JavaScript 版 SICP，全部基于 pair 也许是为了贴近原版 Scheme 语言风格（或者说 Lisp 系风格）？

pair 组成链表，重写 foreach， map, reduce and filter 四个常用操作，以及 unorder list, ordered list, set, tree 等各种高级数据结构

data abstractions 章节通过一个有理数的例子开始，解释怎么构建 Data-abstraction barriers 实现良好的抽象、隔离，这样每一层的变动都不会影响到其他部分，不管是重构还是增加新功能都变得容易很多

两大章看下来，感觉好像学到了好多，但因为没做课后练习，实际印象并不深刻（例如，在写这部分博客时必须回头看书才能写出来点东西）
后面有时间还是做一下课后题比较好

胡思乱想

  • 2018 年：BAKA影 - 【长评】我为什么不喜欢《少女☆歌剧 Revue StarLight》
  • 2024 年：BAKA影 - 【万字分享】如何学会去爱《少女☆歌剧Revue Starlight》

看到一枚时隔六年的回旋镖，up 主在新视频中坦然承认当初因为少歌 bgm 排名比回转企鹅罐高，心生忌妒做出了那期 我为什么不喜欢少歌
六年过去，也许是自然而然，也许是经历了什么，up 主不再掩饰自己对少歌的喜爱，还真诚、勇敢地把自己的想法公开发出来
视频很长，将近 40 分钟，刷了好几次，每次看都佩服 up 主的真诚：不对自己说谎，不对观众说谎
也佩服 up 主的勇敢：承认自己的不足，甚至是通过公开发布视频这种方式

我个人非常喜欢这种分享自己心路历程的资料，尤其是对同一个主题，间隔不算短的时间（六年），可以明显看到一个人的改变
这也是我写博客的原因，把自己当前的想法记录、固定下来，等过几年回头来看，能够清晰地感受到自己的改变（以及一些不变）
2023 年初收拾自己物品时，翻到了高中时语文老师布置的周记作业，看着那些已经被我完全忘记的文字，仿佛在看一个陌生的自己，「原来当时我在想这些东西」、「原来我那时候就有这个苗头」
总之就是非常有意思的事情

说回这个视频，虽然主题是少歌，但视频里插入了几段看似跑题的 up 主个人经历，其中两个地方让我有些共鸣

  • 视频 17:27
    短时间搬家 3.5 次，up 主不敢有一周以上的计划，不敢买实体书
    「我在挣扎，是因为我不想挣扎」、「我不知道我想留住什么，但我想留住一切」

    想到了一句诗「身世浮沉雨打萍」，当然不至于那么惨，但确实会感觉自己就像无根浮萍，虽然每天生活在这里，却不属于这个地方
    中国有所谓 北漂、北上广深，日本有 上京（上东京打拼），如今住在社畜标配鸽子笼里的我，也很少买实体书，因为搬家麻烦，买东西前会考虑出租屋里能否放下、搬家是否方便
    三思而后买，现在家里看起来还算整洁

    但是，我知道我肯定是要搬家的
    将来某一天，我会把这里的所有东西打包，去到其他地方
    下一个地方在哪里？我不知道
    下一个地方之后会不会有下下个地方？我也不知道
    一切都不确定，一切都未知，我就在这看不到方向的洪流中，主动或被动地漂流下去~~~

  • 视频 25:50
    超脱游戏之外，可以随意回溯、改写剧情的玩家，以及无论回溯几次，都无法改变的结局
    明明知道只是游戏设计好的剧情，却会对角色产生愧疚

    一厢情愿的投入过多感情，又一厢情愿的愧疚伤心
    正因为虚幻，所以才纯粹吗…

杂

Daiso カビとり スプレー

浴室通风不及时，长了好多霉点，普通卫生纸根本擦不掉，试了试 Daiso 的除霉菌喷剂，效果非常好
名字就叫 カビとり スプレー，喷出来是泡沫，哪里发霉喷哪里，静置几分钟再用水冲掉，浴室崭新如初

喷剂不能接触皮肤，味道也比较刺鼻，操作过程中注意防护，戴手套、开换气
