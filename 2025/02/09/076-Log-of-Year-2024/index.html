2024 年终总结

本文最后更新于：2025-02-11

创建博客第 7 个年头，终于迎来了第一篇年度总结

从搭建这个博客开始就有写年度总结的打算，但每到年末准备提笔时，却深感自己词汇匮乏、记忆稀薄，以及现实生活贫瘠，似乎没有什么值得特别总结的内容
结果就是想了又想，脑海里的年末总结从阳历新年构思到阴历新年，一直在构思，从来未下笔

不过今年情况发生了变化，得益于（每次拖延半个月）的 12 篇月记，本次 2024 年年度总结可以说是资料丰富
甚至有些丰富过头，有点无从下手的感觉（所以，这就是 2024 年年度总结拖到 2025 年 2 月的理由

思来想去，最终还是计划仿照月记的结构

  1. 输入、输出：2024 看了什么、做了什么
  2. 支出：总结一下 2024 的消费情况
  3. 最后是一些 心态转变

输入

书籍

2024 年阅读技术书籍居多，毕竟半路出家，技术太菜了，需要补补基础

  • 入门

    • 《网络是怎样连接的》、《计算机是怎样跑起来的》、《程序是怎样跑起来的》
      三本日本人写的技术书籍，通俗易懂，值得推荐

      除了这三本日本人写的技术书籍，还看了一本《インフラエンジニアの知識と実務がこれ1冊でしっかりわかる教科書》
      但是看完跟没看一样，一般般，不推荐

  • 高屋建瓴

    • 《人月神话》、《松本行弘的程序世界》、《软件随想录》
      人月神话不用多介绍，1975 年出版的书现在仍然不过时
      第二本书作者松本行弘是 Ruby 语言之父，这本书也挺有意思
      第三本软件随想录作者是 Stack Overflow 创始人之一，文风幽默，非常好读
      我感觉这三本是可以帮助建立对 编程 这个概念总体认识的书籍，有了总体的认识再去深入探索个别领域也许效果会好一些吧

  • Python

    • 《Python 工匠》
      想学习 Python（想了好几年），但由于种种原因，今年只看了这一本 Python 书籍
      在翻看 Python 文档时读到 Ptyhon Documents - Functional Programming HOWTO，简单介绍了一下函数式编程
      之前也偶尔听过这个东西，《松本行弘的程序世界》、《软件随想录》里也都提到了函数式编程，感觉到处都在吹，勾起了我的好奇心

  • JavaScript

    • 《Eloquent JavaScript》
    • 《MDN - JavaScript reference》
      想学 JavaScript（也想了好几年），今年终于付出行动
      Eloquent JavaScript 虽然看完，但后半部分基本是囫囵吞枣，并没有完全看懂（虽说前半部分也不敢说完全理解…）
      比如 正则，实际遇到问题之后再回头看对应章节才知道怎么用，为什么这么用
      还有 元编程 和 异步，直到现在还是没太搞懂这两个东西怎么用，还需要继续研究
      Eloquent JavaScript 虽然覆盖面很广，但还是有许多 JS 知识没讲到，配合 MDN - JavaScript reference 查漏补缺，哪里不会看哪里

  • Regular Expression

    • 《正则表达式30分钟入门教程》
      又又又是想学了好几年的正则表达式，这次是工作需要，倒逼我去光速学习
      掏出这篇在收藏夹里吃灰多年的正则入门教程（严格来说这是一篇文章，不是书籍），结合现实需求，学习成果显著，感觉自己要起飞了
      除了 平衡组/递归匹配 看起来太复杂、现在也用不到，教程里的其他奇技淫巧都完全掌握，可以写出各种魔法正则式救我狗命
      人生苦短，快学正则

  • Elixir

    • 《Learn Functional Programming with Elixir》
    • 《Elixir Documents》
      函数式编程对我来说是完全陌生的领域，但感谢这两个优秀的学习资料，感觉入门没有想象中难
      继续钻研中

  • Others

    • 《はみだしの人類学 ともに生きる方法
      看着本书主要目的是练习阅读日语，内容挺有意思
    • 《つまづきやすい日本語
      内容也可以，看了一半，后面学 Elixir 去了
    • 《Plain Language Guidelines
      想学英语写作，但看了一点，后面都没捡起来

输出

博客

2024 年总共发布 19 篇博客，其中 12 篇是 月记 系列
如果刨去每月水帖，2024 年实际产出 7 篇博客（至于这些文章的质量，也就供自己看一看而已）
虽然想写一些技术文章，但 JavaScript 和 Elixir 经验尚浅，新鲜的知识点过几天回头看时，总感觉这似乎就是理所应当的普通常识了，这点常识真的够资格写成一篇博客吗…

但似乎写出来也无妨啊

所以，2024 年博客的主题就当成 记录自己 吧（

摄影

2024 年在图片管理软件里标记星星⭐的照片有 3000 多张，快门数则是超过了 1w，废片率高达 70%
拍了这么多，姑且感觉摄影技术应该是有点提升（仅限风景摄影，人像会被打）

放一张前段时间拍的照片，满月+飞机，只是焦段太短，月亮和飞机看起来都有点小

现在用起来最顺手的是 光圈优先 + 点测光，偶尔需要控制快门速度时会切纯手动模式
以及，即使我的 A7C2 已经是最轻的全画幅，每次出门前还会纠结要不要背相机，毕竟带上相机就意味着要再加上相机包，负重+++
目前种草 理光 GR3 HDF 中（

自从购入相机，我的博客配图和电脑壁纸都实现了完全自产自销，不用上网搜图，没有版权困扰~
今年过年时突发奇想，把跟朋友出游的照片打印出来，做了几本小相册送给他们
在大家滑滑手机屏幕就能翻阅相册里无数照片的电子时代，我感觉这样看得见、摸得着、只有寥寥几张照片的实体相册反而变得珍贵了起来

自认为这才是比较有趣的礼物😋（至于收到相片的人觉得是否有趣就是另一个问题了）

关于打印照片，我是用 Yodobashi 或 Biccamera 的自助相片打印机，50 日元/张，价格不算贵
但这个打印机虽然挂着 富士 的牌子，实际打印效果只能说….能打而已
主要问题是色彩不准，和电脑屏幕上看到的偏差较大，同时打印机还会擅自加曝光，电脑上曝光正常的图片，打印出来亮部就有可能过曝
不知道有没有其他比较靠谱（并且实惠）的照片打印服务

支出

“钱都去哪了？”，自从我养成记账的习惯后，就再也没有这个困扰了，假的，每个月回顾账本都会发出疑问：明明感觉没有花很多啊，为什么加起来这么高？
即使每次月底结算都会让人两眼一黑，记账总归还是有一些好处的，最起码可以让我清楚地了解自己的钱都花在了哪里，吃？喝？玩？乐？（都不是，是房租哒）

2023 年初到日本，第一年支出波动较大，不具备参考意义，生活稳定下来后的 2024 年支出情况就可以拿出来稍作分析了
这样做一方面是对去年消费状况的回顾，基于数据分析自己的消费模式到底是什么样子，而不只是模糊的感觉
另一方面，这同时是给我自己提供一个客观、可衡量的标准，方便评价未来的开销情况

顺便，虽然每个人的消费观念不同，也许没办法直接比较，但如果有人对日本生活成本好奇的话，这里的例子也许可以提供一些参考

先介绍基本情况：一人一猫，东京打工，没什么烧钱的兴趣爱好，比较宅，去年没有旅行
支出总额：2024 年总支出约 300 万日元

各类支出占比如下

  • 衣，不到 4%
    平均每月不到 1w 日元，对衣服完全没追求，MUJI、优衣库，再加上 BookOff 解决所有

  • 食，约 20%
    平均每月 5w 日元，包含吃饭、饮料、零食（很少吃）
    对吃饭同样没什么追求，懒于钻研厨艺，出勤的话午饭在路边随便吃大概 1k，其他餐在家随便糊弄一下（倾向选择耗时少的半成品），每周可能跟朋友聚餐一到三次
    目前 5w 日元/月，感觉算是中规中矩的水平，有朋友点外卖每个月吃饭花 10w，在网上也看到过一家三口每个月 4w 日元的案例
    吃饭这部分丰俭由人吧

  • 住，约 50%
    最大的一块支出，包含房租、水电、宽带、手机卡费用
    这些雷打不动的固定开销占据了全年支出的一半，偏偏又都是没办法削减的成本
    本人房租在几个朋友中算是偏贵的一档，原因是日本出租屋养宠物需要房东同意，再加上外国人租房本就是困难模式，难上加难，可供选择的房源非常有限
    最终只能选择 加钱…找到了现在这套通勤还算便利的当代社畜标配单身公寓（迷你鸽子笼）
    虽然后续一直有搬家的想法，但碍于搬家成本（搬家费+房租头金）太高，而且万一运气不好，搬到下一家发现还不如现在的鸽子笼，可是后悔都来不及
    总之，将就吧…

  • 行，约 4%
    在日本交通费用是可以记入账单的项目，最便宜的 JR 起步价是 148 日元（2023 年上涨 10 日元后的价格），出一趟远门往返路费超过 1k 日元并不罕见
    不过社畜有个福利(?)是公司报销通勤交通费，买一张 定期券(交通月卡) 说不定周末出门也能稍微蹭一下，便宜几块钱（
    本人总体比较宅，周末出门不多，所以这部分支出大头应该是通勤交通费（虽说有报销，但分开计算比较麻烦，干脆都记在账上）

  • 电子产品，约 12%
    看起来有点高（实际也有点高），主要支出是买了一支 Sony GM 50mm F1.4 镜头，占据了本年电子产品消费的 50% 以上
    刚才明明说过 没什么烧钱的兴趣爱好，但摄影器材好像对钱包并不友好（

    狡辩的话术已经想好了，诸如 早买早享受，晚买也没折扣、摄影器材是肯定保值的，而且使用时间越久平均成本反而越低、这是一笔固定资产投入，应该按使用年限把成本均摊到每年账本里（

  • 娱乐，约 5%
    包含看电影、展览、各种活动、周边商品之类
    电影可能每个月看两三场，二次元周边不会买大件（手办之类），因为家里放不下
    二次元的钱真好赚，在我这里没那么好赚

  • 其他，约 5%
    包含猫咪开销，猫粮、猫砂之类，以及平时自己头疼脑热、定期牙科诊所之类的医疗费用，再加上其他林林总总的细碎开支，都算在这里面

    今年没什么大病，万幸

全年 300w 日元，平均 25w 日元/月，回顾下来感觉消费水平还算比较合理，希望新的（已经过去 1.5 个月的）一年继续保持~

心态转变

自从 2023 年年初来到日本，目前也差不多过去两年了

第一年： 焦虑

现在回头看来，第一年自己心态比较急躁，初来乍到以我的哑巴日语水平，日常生活实际比较吃力，再加上自己跨行业跑来做 IT，编程水平可以说完全没有…
当时比较焦虑怎么快速提高技术和日语水平，遇到一个人就想问问他们是怎么上手的，有没有什么给新人的建议，听谁说话感觉都挺有道理
但 技术 和 日语 这两方面又不是一朝一夕就能提高的事情，现在回过头看，当时的自己就像无头苍蝇，整天焦虑 如何做，但实际付出的行动却比较少…

第二年： 躺了 顺其自然

直到第二年，才感觉自己心态放松了一些，不会太过急于求成，允许自己放慢一些步调，按照自己的速度稳步推进
日语方面，感觉有比较明显进步（也许有第一年积累的功劳？），尤其是听力水平，能听懂更多了，口语姑且也有进步，虽然还是散装日语水平
技术方面，开始静下心学习 JavaScript，顺便搞懂了正则表达式，还发现 Elixir 这个看起来非常有趣的东西

现在回想起来，有些之前看起来好厉害的前辈现在感觉也就那样，似乎挺有道理的建议现在看来也不一定对
更重要是，总感觉日本 IT 行业整体水平似乎有些差…（也可能是因为我接触到的都是低水平项目？）

人为什么要工作？.jpg

2024 下半年进入了一个纯正日式开发项目
为了显得流程规范而刻意做一堆无用功、面向 Excel 编程、在操蛋的事情上精益求精，等等等等等…
这些日式 IT 项目里司空见惯的事情给我狠狠地上了一课，让我体会到了什么叫作 工作的虚无

从进入项目开始，到经过半年左右，一些问题始终在我的脑海里徘徊：
这份工作有什么意义？
人生有什么意义？
我自己想过什么样的人生？

毫无意义的工作内容刺激我开始思考这些之前忽视的烂大街问题，虽然思考来思考去也得不出什么像样的答案，但思考过程中确实有一些事情在逐渐变清晰

  • 例如，很多人追求的名牌、奢饰品、豪车别墅，这些所谓 世俗意义上的成功 对我来说似乎没什么吸引力
    300w 日元就可以覆盖一整年基本开销的情况下，为工作内耗太多好像没有什么意义
  • 以及，如果我以后不再需要为签证困扰，甚至更进一步，不再需要为金钱困扰，到那时我会怎样安排自己的时间？要去做些什么呢？
    关于这个问题，目前只有一些似是而非的模糊方向，没有明确答案
  • 说到时间，我现在感觉 属于自己的时间 是需要用心守护的，比起耗费精力占一点小便宜，或是去无聊的聚会认识一堆表面朋友，我宁愿把这些时间花在看动画、去河边散步、甚至是睡大觉，后者才是真正让时间为自己所用

所以，2024 年的心态变化总结下来就是

  1. 从活力满满的萌新进化（or 退化？）为上班如上坟的社畜，人为什么要工作？.jpg
  2. 我想活出怎样的人生？，持续思考中
  3. 吝啬 属于自己的时间

结语

不管遗憾还是满足，亦或是两者皆有，2024 年总归已经过去了
回顾就到此为止，接下来对（已经过去 1.5 个月的）2025 年做一些展望吧

  1. 为了离我理想的生活更近一步，希望 2025 年继续精进技术和日语
  2. 希望自己和周围人没什么大灾大病，身体健康
  3. 世界和平

除了 提高技术和日语 尚在人力范围内，其他都是胡扯（

所以，一句话总结： 提高专业能力 & 照顾好自己（和猫）
就酱

それでは、ちょっと遅いけど、良いお年を〜
